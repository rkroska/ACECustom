# ACE Web Portal - ClientApp Documentation

This document provides a technical overview of the frontend architecture, design standards, and localized component structure for the ACE Web Portal.

> [!IMPORTANT]
> **Build Requirement**: Anytime a file in the `ClientApp` directory is modified, you **MUST** execute `npm run build` from the `ClientApp` folder to update the production assets. The `ACE.Server` specifically serves the `dist/` folder generated by this command.

## 🎨 Design Philosophy & Style

The application implements a **modern, high-density dashboard aesthetic** optimized for power users (Server Admins and Game Masters).

- **Theme**: Dark-mode primary palette using Tailwind's `neutral-900` and `neutral-950`.
- **Accents**: 
  - **Blue (`blue-600`)**: Primary actions, active routes, and global highlights.
  - **Purple (`purple-500`)**: Metadata links and complex property mappings.
  - **Green/Red**: Real-time session status (Online/Offline) and transaction feedback (Success/Error).
- **Rich Aesthetics**: 
  - High-fidelity `backdrop-blur-xl` for card surfaces.
  - Interactive scale transitions and custom "glow" effects for interactive elements.
  - Custom-themed scrollbars for consistent cross-browser density.
- **Typography**: Heavily utilizes high-contrast font weights (`font-black`) for hierarchy and mono-spaced fonts for technical IDs (GUID, WCID).

## 🏗️ Technical Architecture

The frontend follows a **Service-Oriented Architecture (SOA)** pattern to ensure scalability and ease of testing.

- **Routing**: Powered by `react-router-dom` using a standardized `:guid/:tab?` pattern. Supports both personal character views and administrative player lookups while maintaining context.
- **Service Layer**: All communication is consolidated into the `api` service (`src/services/api.ts`). This client handles:
    - **Secure Credentials**: Uses `credentials: 'include'` to support `HttpOnly` server cookies.
    - **Request Resilience**: Optimized with non-JSON fallback parsing to prevent UI crashes on server-level error pages. 
    - **Global Interceptors**: Handling for `401` (Unauthorized) and `503` (Portal Disabled) states.
- **Global State**: Managed via **Zustand** (`src/store/useAuthStore.ts`) for authentication and session management.
    - **In-Memory Store**: All sensitive session state is in-memory only. 
    - **Bootstrap Sync**: The application automatically validates and restores sessions from the secure server cookie on every load via an asynchronous bootstrapping sequence.
- **Core Stack**: React 18+, TypeScript 5+, Vite, Tailwind CSS, Lucide Icons.

## 📂 Directory Structure

The project uses a **Feature-Based Reorganization** strategy to house components based on their activity and general purpose:

- **`src/components/`**: Route-level "Main" components (e.g., `CharacterList.tsx`, `PropertyExplorer.tsx`).
- **`src/components/character/`**: Character activity sub-components (Stats, Skills, Inventory, etc.).
- **`src/components/admin/`**: Administrative tool sub-components (`EnumIndexItem.tsx`, `EnumValueListItem.tsx`, `ServerParamListItem.tsx`).
- **`src/components/common/`**: Shared UI primitives and layout elements (`Modal.tsx`, `Sidebar.tsx`, `PageHeader.tsx`, `TabButton.tsx`).
- **`src/types/`**: Single source of truth for all entity interfaces and API metadata.
- **`src/services/`**: Centralized logic for network and utility operations.

## 📋 Module Breakdown

### 🔘 Activity Hubs (Nav-Level)
- **`CharacterList`**: Grid view of account-scoped characters.
- **`CharacterDetail`**: Central routing hub for character deep-dives. Supports dynamic tab switching without losing page context.
- **`PlayerList`**: High-performance admin directory using `useDebounce` for real-time global character searches.
- **`PropertyExplorer` / `LookupTables`**: Content discovery tools with deep-linking support for server enums. Includes advanced error handling with localized retry functionality for data retrieval failures.

### 👤 Character Activity Sub-Components
- **`CharacterStats`**: Decomposed view of Attributes and Ratings. Logic is powered by server-side snapshots.
- **`CharacterInventory`**: Recursive tree-search and composite rendering (PNG generation with overlays/underlays).
- **`CharacterSkills`**: Filterable list using server-authoritative logic.
- **`CharacterStamps`**: Paginated view of quest completions with localized "Copy All" functionality.

## 🛠️ Interaction Patterns
1. **Speed-to-Game**: Extensive use of command template generation (Copy-to-Clipboard) for administrative efficiency.
2. **Optimistic Feedback**: Immediate UI updates for forced logouts and status changes.
3. **Adaptive UI**: High-density layouts that automatically adjust based on the current window size and sidebar state.
